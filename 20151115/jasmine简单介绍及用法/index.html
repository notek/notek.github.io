jasmine简单介绍及用法 2015-11-15

jasmine是什么？

jasmine是一个基于BDD的javascript开发测试框架。

jasmine是一个独立运行的测试框架，不依赖其他的框架，可用于nodejs端和纯前端环境中。

下载jasmine

本文所有的例子均在纯前端的测试环境中，所以在开始之前请到这里下载jasmine。

本文使用的是jasmine-standalone-2.3.4.zip。

基本概念

Suite

Suite是一个测试组，可以简单理解为一组测试。测试组可以有多个，也可以相互嵌套。

Spec

Spec是一个单个测试，定义于某个测试组中。可以同时存在多个。

Expectation

Expectation是一个断言，代表着程序运行期待的结果。

基本方法

上面简单说了3个基本概念，下面介绍几个基本也是最重要的方法。

describe

describe用来定义一个测试组，接受两个参数：

describe(string, fn)

string: 定义一组测试的描述
fn: 定义一组测试的行为

/*** usage example ***/
describe('the test group', function() {
    //some code.
});
it

it方法用来定义单个测试，也接受两个参数：

it(string, fn)

string: 定义一个测试的描述
fn: 定义一个测试的行为

/*** usage example ***/
it('the test', function() {
    //some code.
});
expect

expect方法用来定义一个断言，接受一个参数：

expect(actual)

actual: 代表一个实际的值

/*** usage example ***/
expect(100)...
matcher

matcher不是一个具体的方法，而是代表一些期望行为的判断。必须配合expect方法使用。主要的matcher有以下几种：

  • toBe: 使用 === 判断实际值和期望值
  • toEqual: 判断两个值/对象是否相等
  • toBeDefined: 判断一个值不是undefined
  • toBeUndefined: 判断一个值是undefined
  • toBeMatch: 使用正则匹配判断实际值是否包含正则表达式定义的查询
  • toBeNull: 判断一个值是不是null
  • toBeTruthy: 判断一个值是否为true,会按照javascript转换规则进行比较转换
  • toBeFalsy: 判断一个值是否为false,会按照javascript转换规则进行比较转换
  • toContain: 判断数组是否包含某个值
  • tobeLessThan: 对实际值和期望值进行小于判断
  • toBeGreaterThan: 对实际值和期望值进行大于判断
  • toThrow: 判断一个函数是否抛出了异常
  • toBeCloseTo: 判断两个数是否足够接近

jasmine项目结构简洁

现在打开之前下载的jasmine，会看到以下项目结构：

jasmine
   |-- lib
   |-- MIT.LISENCE
   |-- spec
   |-- SpecRunner.html
   |-- src
  • lib：存放的是所有的基础服务文件
  • MIT.LISENCE：协议声明
  • spec：测试文件
  • SpecRunner.html：测试结果输出页面
  • src：被测试的文件

基本的例子

现在来写一个简单的比较基本类型值的测试，在spec目录下新建文件basicCompareSpec.js，加入以下内容：

describe('比较基本类型值是否相等', function() {
    var a = 10,
        b = 11;

    it('看看a是不是等于b', function() {
        expect(a).toEqual(b);  //还记得toEqual这个matcher吗？它是用来比较两个值是否相等的
    });
});

然后在SpecRunner.html文件中引用该文件：

<!-- include spec files here... -->
<script src="spec/SpecHelper.js"></script>
<script src="spec/PlayerSpec.js"></script>
<!-- 在这里加入新建的basicCompare文件 -->
<script src="spec/basicCompareSpec.js"></script>

最后运行SpecRunner.html文件，就能看到结果提示：

测试不通过，并且提示了具体问题在哪。这时候在把b的值改为10就能通过测试了。

稍微复杂一点的例子

现在来做一个比较实际的例子，可能会比上面的例子稍微复杂那么一丢丢，限制用户必须输入且必须是数字。

继续在新建一个叫做mustBeNumberSpec.js的文件：

describe('must be a number'. function() {

    //假设userInput为用户输入
    var userInput = function() {
        return 15;
    };

    it('测试用户是否输入且是不是数字', function() {
        //先检查是否输入
        expect(userInput()).toDndefined();
        //再检查是不是数字
        expect(userInput()).toMatch(/^\d*$/g);
    });

});

这个例子其实也很简单，在实际应用中，可能更多的方面会要求测试异步代码，在jasmine中测试异步代码其实也很简单，但是会涉及到两个前面并没有说过的方法。

beforeEach和done方法

beforeEach是用来定义在一个测试组中所有Specs开始之前需要初始化的东西。

done是一个可选方法，在beforeEach/it等方法中作为回调函数的参数存在，在测试异步例子的时候这很重要。如果忘记了调用done方法，那么spec会直接等待5s，然后就会因为超时而直接退出，我么将得不到任何有用的信息。

顺便提一句，在jasmine中设置超时时间是通过jasmine.DEFAULT_TIMEOUT_INTERVAL设置，默认就是5s。

ok，现在我们定义一个异步方法，然后测试它。

新建asyncSpec.js文件：

describe('测试异步支持', function() {

    var val;

    //定义异步方法
    beforeEach(function(done) {
        setTimeout(function() {
            val = 100;
            //调用done
            done();
        }, 1000);
    });

    //测试异步方法
    it('检测通过异步方法设置的val变量的值是否是10', function(done) {
        expect(val).toEqual(10);
        //在it(spec)里面也必须调用done
        done();
    });

});

然后现在执行SpecRunner.html文件，我们可以看到，页面在等待了1s之后输出了测试通过的结果。
