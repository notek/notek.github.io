插件推荐(排名不分先后)

AdvancedNewFile

AdvancedNewFile是一个用来创建文件的插件，默认情况下创建文件会使用command + n然后command + s保存，有时候还会选择保存的路径等等，比较繁琐，使用AdvancedNewFile可以简化这一过程并且默认会将文件保存到当前目录，也可以在创建文件的时候就带上路径。

AdvancedNewFile使用command + option + n来创建文件。

AutoFileName

AutoFileName是一个能帮助你在写代码的时候自动生成路径的插件，它会根据当前文件所处的位置使用相对路径来生产文件的插件，使用起来很方便，没有快捷键。

效果图：

BracketHighlighter

BracketHighlighter是一个高亮显示括号的插件，默认显示方式为underline，但是不是很明显，可以将package Settings > Bracket Highlighter > Bracket Setting - Default文件打开并复制一份，保存到package Settings > Bracket Highlighter > Bracket Setting - User中，然后找到bracket_styles选项，将其中的default选项的style属性修改为outline，其支持4种高亮方式:underline/outline/highlight/solid，一般选择修改为outline就比较明显了。

效果图：

Color Highlight

Color Highlight是一个实时显示颜色值的插件，前端吐血推荐！它能帮助你实时查看颜色，并且自带一个颜色选择器，在你的光标移动到对应的颜色值上时，背景色将会改变为颜色值所对应的颜色。

效果图：

ColorPicker

ColorPicker是一个颜色选择插件，前端吐血推荐！使用快捷键command + shift + c便能触发颜色选择，将弹出一个颜色选择器，选择好颜色之后选择pick就能应用到对应的位置。

效果图：

DocBlockr

DocBlockr是一个能自动帮你生成块注释的插件，并且会根据当前函数生成一些初始信息，能帮你节约不少时间。

使用方式也很简单，在你需要生成注释的函数的上面一行键入/**然后敲回车键，就能帮你生成块注释信息。

Emmet

Emmet是一个地球人都知道的神器，前端吐血跳楼推荐级神器。是大名鼎鼎的zen coding的延伸版。

虽然是人尽皆知的神器，还是说明一下它的作用，没错，就是帮你快速生成html、css等代码，不赘述了，想了解的去Emmet官网查看使用方法。

Function Name Display

Function Name Display是一个能在状态栏帮助显示当前位于哪个函数内的插件，作用非常简单。

SideBarEnhancements

SideBarEnhancements是一个快捷键增强插件，使用最广泛的就是用来定义浏览器预览文件。

打开Package Setting > Side Bar > Key Bindings - User即可自定义浏览器预览快捷键，如下代码定义chrome所示：

[
    //borwser preview.
    {
        "keys": ["f2"],
        "command": "side_bar_files_open_with",
        "args": {
            "paths": [],
            "application": "/Applications/Google Chrome.app",
            "extensions": ".*"
        }
    }
]

keys字段表示绑定的快捷键，command字段表示关联的程序，application表示要打开的软件，最后extensions表示将要使用软件打开何种扩展文件，.*表示所有文件。

自定义代码片段(snippet)

sublime text3的自定义代码片段片段也很牛逼，虽然现在有了Emmet一些常用的片段都已经帮我们定义好了，但是有时候免不了需要自己定义一些带有浓重的程序员色彩的个人代码片段。

其实在sublime text3中自定义代码片段是一件很简单的事，现在让我们开始。

首先点击Tools > New Snippet，sublime会帮我们自动生成一个代码片段模版：

<snippet>
    <content><![CDATA[
        Hello, ${1:this} is a ${2:snippet}.
    ]]></content>
    <!-- Optional: Set a tabTrigger to define how to trigger the snippet -->
    <!-- <tabTrigger>hello</tabTrigger> -->
    <!-- Optional: Set a scope to limit where the snippet will trigger -->
    <!-- <scope>source.python</scope> -->
</snippet>

里面有3个字段(两个被注释掉了)，代表什么意思呢？

  • content： 这个字段里面定义的就是我们需要当成代码片段的代码
  • tabTrigger： 这里面定义触发的关键字
  • scope： 定义在哪种类型的文件中有效，可以定义多个

其实除了这三个字段外，还有一个叫做description的字段，用来定义触发自定义代码片段的时候的描述信息。

现在，我们定义一个简单的css reset代码片段：

<snippet>
<content><![CDATA[
body {
    margin: 0;
}
]]></content>
    <tabTrigger>crt</tabTrigger>
    <description>我是css reset代码片段</description>
    <scope>source.css, source.css.less</scope>
</snippet>

例子代码中，为了能在less中可用，我加入了less文件的作用域source.less，具体的作用域请看这里。

ok，保存文件为reset.sublime-snippet，现在可以在sublime text3中来检查我们刚才定义的snippet是否有效，输入ctr1并且按下tab键：

crt是我本地定义的，crt1是刚才定义的

可以清晰的看到，description字段定义的描述会在这里显示，这时候在选择crt1按下回车，就会生成我们在代码片段里面定义的代码了。

按照这种方法，可以定义几乎各种程序语言的代码片段，真是爽到爆了。

说到这里，顺便提一下，如果觉得tab键不喜欢，可用重新定义的，打开Sublime Text > Preferences > Key Bindings - User，定义以下代码：

[
    { "keys": ["快捷键"], "command": "auto_complete" }
]

快捷键处请自己定义。然后保存就可以使用，不用重启。
